【トラブルシューティング：azコマンドが認識されない】

事象: インストール直後に az コマンドを実行すると ObjectNotFound エラーが発生。

<img width="1788" height="218" alt="スクリーンショット 2026-05-22 133945" src="https://github.com/user-attachments/assets/a37c40df-fdd0-482f-801e-30bffbae16be" />


原因: インストールによって更新された環境変数が、既存のシェルセッションに反映されていなかった。

対策: シェル（PowerShell）を再起動するも解決せず、再度インストールを行い解決。

Get-WmiObject -Class Win32_Product | Where-Object { $_.Name -like "*Azure CLI*" } | ForEach-Object { $_.Uninstall() }
Invoke-WebRequest -Uri https://aka.ms/installazurecliwindows -OutFile .\AzureCLI.msi
Start-Process msiexec.exe -Wait -ArgumentList '/I AzureCLI.msi'
